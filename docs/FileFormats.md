# YM2149 File Formats Specification

This document consolidates the specification of all asset formats, inputs, and build target outputs used in the Lokey YM2149 audio toolchain. While the primary target is currently the Atari 7800 console (1.789773 MHz), the binary specs and compiler tools remain platform-agnostic, supporting any YM2149 or AY-3-8910 platform (such as MSX, ZX Spectrum, Amstrad CPC, or Atari ST) by adjusting clock ratios and timing.

---

## 1. File Extensions Quick Reference

| Extension | Category | Name | Direction | Description |
| :--- | :--- | :--- | :--- | :--- |
| **`.ym`** | Input | YM Music Dump | Read (`ym-song`) | Original Atari ST YM5/YM6 register streams. |
| **`.json`** | Input | YM Source | Read (`ym-sfx`/`ym-song`) | Hand-authored sound effect or music sequence source files. |
| **`.csv`** | Input | AYFX CSV Export | Read (`ym-sfx`) | Comma-separated values (5-column hex registers) exported from AYFXedit. |
| **`.afx`** | Input | AYFX Effect | Read (`ym-sfx`) | Single binary sound effect file generated by AYFXedit. |
| **`.ysg`** | Output | YM Song Binary | Write (`ym-song`) | Deduplicated pattern-based music data optimized for 6502 replayers. |
| **`.yfx`** | Output | YM Sound FX Binary | Write (`ym-sfx`) | 5-byte fixed-width binary format optimized for rapid VBI overrides. |

## 2. Music Format Specification (`.ysg`)

The `.ysg` (YM Song) format stores pattern-compressed register data optimized for low CPU overhead on the 6502 CPU and bank-switched cartridges.

### Header Structure
The file begins with a **4-byte fixed header**:

| Offset | Size | Name | Description |
| :--- | :--- | :--- | :--- |
| **0** | 1 byte | `PatternSize` | Number of frames per pattern block (e.g. 64, 96). |
| **1** | 1 byte | `NumUnique` | Total number of unique pattern blocks stored (Max 255). |
| **2** | 1 byte | `SeqLen` | Total number of pattern entries in the sequence table (Max 255). |
| **3** | 1 byte | `LoopPattern` | Index in the sequence table to loop back to (255 = no loop). |

### Sequence Table
Immediately following the header:
* **Size**: `SeqLen` bytes.
* **Format**: Each byte is an index (`0` to `NumUnique - 1`) specifying the playback order of the unique patterns.

### Pattern Offset Table
Following the sequence table:
* **Size**: `NumUnique * 4` bytes.
* **Format**: Array of 32-bit little-endian pointers.
* **Reference**: Offsets are relative to the start of the **Pattern Data** block, preventing 64KB ROM boundary wraps for large songs.

### Pattern Data
The final block contains the compressed frame streams:
* **Format**: Bitmask Deltas.
* **Frame Structure**:
  1. **Mask (2 bytes)**: 16-bit little-endian value. Bits 0–13 represent YM2149 registers 0–13. A `1` indicates the register is modified; `0` carries the previous frame's value forward.
  2. **Payload (Variable)**: The register values corresponding to `1` bits in the mask, packed sequentially.
* **Pattern Independence**: The first frame of every pattern block is unconditionally encoded with a full `0x3FFF` mask (resetting all 14 registers), enabling $O(1)$ pattern seeking, looping, and clean SFX channel recovery.

## 3. Sound Effects Ingestion & Target Specs

Sound effects are authored as CSV, JSON, or AFX files, and compiled to `.yfx` target binaries.

### Input Format A: AYFX CSV
Comma-separated values generated by the **AYFXedit** utility:
* **Format**: `ToneEnable,NoiseEnable,TonePeriod,NoisePeriod,Volume`
* *Note: Enables are active-high in CSV (1 = ON, 0 = OFF), opposite of the physical register bits.*

```csv
0,1,0x8a8,0x1f,0xf
0,1,0x8a8,0x1c,0xe
```

### Input Format B: JSON Source
Designed for hand-authoring and delta-encoded (omitted fields carry forward):
```json
{
  "name": "sfx_laser",
  "sourceClock": 2000000,
  "sourceHz": 50,
  "priority": 128,
  "frames": [
    { "toneEnable": true, "tone": 450, "volume": 15 },
    { "tone": 500, "volume": 12 },
    { "volume": 0 }
  ]
}
```

### Input Format C: AYFX Binary (`.afx`)
Flag-encoded frame updates:
* **Flag Byte**:
  * **Bits 0–3**: Volume (`0–15`).
  * **Bit 4**: Disable Tone (`1` = OFF, `0` = ON).
  * **Bit 5**: Change Tone (if set, `2` bytes of LSB,MSB tone period follow).
  * **Bit 6**: Change Noise (if set, `1` byte of noise period follows).
  * **Bit 7**: Disable Noise (`1` = OFF, `0` = ON).
* **End Marker**: Byte sequence `0xD0 0x20`.

### Target Output Format (`.yfx`)
Each audio frame is exactly **5 bytes** wide:

| Byte Offset | Size | Name | Description |
| :--- | :--- | :--- | :--- |
| **0** | 1 byte | `PitchLow` | Low 8 bits of the channel's tone period (R0/R2/R4). |
| **1** | 1 byte | `PitchHigh` | High 4 bits of the tone period (R1/R3/R5). |
| **2** | 1 byte | `Volume` | 4-bit channel volume amplitude (R8/R9/R10). |
| **3** | 1 byte | `Control` | Mixer enables and noise period (see bit allocation below). |
| **4** | 1 byte | `Duration` | Hold duration in VBI ticks (specifies length of hold). |

#### Control Byte Bit Allocation (Byte 3)
* **Bit 0**: Tone Enable (`1` = ON, `0` = OFF).
* **Bit 1**: Noise Enable (`1` = ON, `0` = OFF).
* **Bit 2**: Reserved.
* **Bits 3–7** (5 bits): Global Noise Period divider (`0` to `31`).

## 4. Unsupported YM Variants

Some legacy chiptune files using the `.ym` extension contain structure variants that are fundamentally incompatible with single-chip register streaming:

* **`MIX1` (Stereo 2-Chip)**: A stereo format designed by Leonard/STSound that streams to two parallel YM2149 chips simultaneously (6 channels total). Because the physical Atari 7800 expansion board only features a single YM2149 chip, stereo 6-channel playbacks are unsupported.
* **`YMT1` / `YMT2` (Tracker Pattern Bytecode)**: A compressed tracker format that relies on custom 68000/Z80 assembly runtime macros and software synthesis rather than raw register streams. These are unsupported because the 6502 replayer targets pure register-streaming payloads.
